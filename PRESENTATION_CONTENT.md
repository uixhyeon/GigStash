## 📑 슬라이드 4: 1-가. 문제 정의

### 현재 문제점

**대규모 행사장의 짐 보관 문제**

1. **불편한 짐 보관**
   - 공연, 스포츠, 페스티벌 등 대규모 행사 참석 시 짐 보관 어려움
   - 기존 보관함: 위치 파악 어려움, 사용 가능 여부 불명확

2. **비효율적인 운영**
   - 행사 주최측: 짐 보관 관리의 어려움
   - 수작업 관리로 인한 시간 낭비 및 오류 발생

3. **기사 업무 관리 문제**
   - 기사의 스케줄 및 업무 현황 파악 어려움
   - 실시간 보관함 상태 확인 불가

## 📑 슬라이드 6: 1-다. 관리자 페이지 기획 개요

### 관리자 페이지 주요 기능

1. **대시보드 (AdminMain)**
   - 전체 공지사항 관리
   - 당일 보관함 현황 (사용중/예약중/미배정/이용가능)

2. **행사 관리 (EventView)**
   - 행사 등록/수정/삭제
   - 행사별 예약 현황
   - 차량 배차 관리

3. **예약 관리 (ReservationView)**
   - 예약 목록 조회/검색/필터
   - 예약 상태 관리 (활성/완료/취소)
   - 예약 상세 정보 확인

4. **모니터링 (MonitoringView)**
   - 실시간 차트 및 통계
   - 행사별/기간별 데이터 분석

## 📑 슬라이드 16: 2-사. 정보구조 설계

### ERD (Entity Relationship Diagram)

**주요 엔티티**

1. **CUSTOMER (고객)**
   - 고객 정보, 멤버십, 예약 이력

2. **RESERVATION (예약)**
   - 예약 정보, 상태, 보관함 번호

3. **EVENT (행사)**
   - 행사 정보, 일정, 장소

4. **LOCKER (보관함)**
   - 보관함 번호, 상태, 위치

5. **VEHICLE (차량)**
   - 배차 정보, 기사 배정

6. **WORKER (기사)**
   - 기사 정보, 근무 이력

**관계**
- CUSTOMER 1:N RESERVATION
- EVENT 1:N RESERVATION
- LOCKER 1:N RESERVATION
- WORKER 1:N VEHICLE
- EVENT 1:N VEHICLE
